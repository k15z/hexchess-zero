The neural network is defined in `training/model.py`. It's a convolutional residual network inspired by Leela Chess Zero and KataGo.

## Input tensor

The input is a $(19, 11, 11)$ tensor — the 91-cell hex board embedded in an 11×11 grid with zero-padding for invalid cells:

| Channels | Content |
|----------|---------|
| 0–11 | Piece planes (6 types $\times$ 2 colors) |
| 12 | Side to move |
| 13 | Fullmove number |
| 14 | Halfmove clock |
| 15 | En passant |
| 16 | Repetition count |
| 17 | Validity mask |
| 18 | In-check flag |

## SE residual blocks

Each block follows the **Squeeze-and-Excitation** pattern (Leela Chess variant):

<Mermaid chart={`
graph LR
    X["x"] --> Conv1["Conv 3×3"] --> BN1["BN"] --> ReLU1["ReLU"] --> Conv2["Conv 3×3"] --> BN2["BN"] --> SE["SE Module"] --> Add["+ x"] --> ReLU2["ReLU"] --> Out["output"]
    X --> Add
`} caption="SE Residual Block: skip connection with squeeze-and-excitation" />

The SE module learns per-channel scaling:
1. Global average pool → $(B, C)$
2. FC → ReLU → FC → split into **scale** (sigmoid) and **bias**
3. Output = $\text{scale} \odot x + \text{bias}$

This gives each channel a learned, input-dependent scaling — the network can dynamically emphasize or suppress features based on the global position.

## KataGo-style global pooling

At blocks 3 and 6, an additional **global pooling branch** injects board-wide context:

1. A parallel 1×1 conv computes `pool_channels` features
2. Three pooling operations: mean, scaled mean, max
3. The pooled features are projected to a per-channel bias added before the second convolution

This lets the network reason about global properties (material balance, king safety, piece counts) while still operating with local 3×3 convolutions.

## Policy head

$$
\text{Conv}_{1\times1}(192 \to 8) \to \text{BN} \to \text{ReLU} \to \text{Flatten} \to \text{Linear} \to 4206 \text{ logits}
$$

Outputs logits over the deterministic move index table. During training, the target is the MCTS visit count distribution (after temperature). During inference, legal moves are masked and softmax is applied.

## Value head

$$
\text{Conv}_{1\times1}(192 \to 32) \to \text{BN} \to \text{ReLU} \to \text{GAP} \to \text{Linear} \to 3 \text{ WDL logits}
$$

Outputs **Win/Draw/Loss** logits rather than a single scalar. This provides richer training signal (the network learns to distinguish wins from draws) and converts to a scalar via $v = W - L$ for MCTS backup.

## Training loss

The combined loss function:

$$
\mathcal{L} = \underbrace{\text{CE}(\pi_\theta, \pi_{\text{MCTS}})}_{\text{policy loss}} + \underbrace{\text{CE}(v_\theta, z)}_{\text{value loss}} + \underbrace{c \|\theta\|^2}_{\text{L2 regularization}}
$$

where $\pi_{\text{MCTS}}$ is the search policy target, $z$ is the game outcome (WDL), and $c = 10^{-4}$.

## Architecture parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `num_residual_blocks` | 10 | Depth of the residual tower |
| `num_filters` | 192 | Width of convolutional layers |
| `se_channels` | 48 | SE bottleneck width |
| `global_pool_channels` | 32 | Global pooling feature width |
| `global_pool_blocks` | (3, 6) | Which blocks get global pooling |
| `policy_channels` | 8 | Conv channels in policy head |
| `value_channels` | 32 | Conv channels in value head |
