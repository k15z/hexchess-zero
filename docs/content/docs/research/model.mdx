The production network is defined in `training/model.py`. The current architecture is the `global_pool_se` variant chosen from the experiments in `training/experiments/RESULTS.md`: an SE residual tower with KataGo-style global pooling plus auxiliary heads that help training stay grounded on game length and short-horizon outcomes.

## Input tensor

The input is a **side-to-move framed** $(22, 11, 11)$ tensor: the 91-cell hex board embedded in an 11×11 grid with zero-padding for invalid cells, plus metadata planes for repetition, check state, and recent-move context. See [Board](/docs/engine/board#tensor-encoding) for the full channel breakdown.

## SE residual blocks

Each block follows the **Squeeze-and-Excitation** pattern (Leela Chess variant):

<Mermaid chart={`
graph LR
    X["x"] --> Conv1["Conv 3×3"] --> BN1["BN"] --> ReLU1["ReLU"] --> Conv2["Conv 3×3"] --> BN2["BN"] --> SE["SE Module"] --> Add["+ x"] --> ReLU2["ReLU"] --> Out["output"]
    X --> Add
`} caption="SE Residual Block: skip connection with squeeze-and-excitation" />

The SE module learns per-channel scaling:
1. Global average pool → $(B, C)$
2. FC → ReLU → FC → split into **scale** (sigmoid) and **bias**
3. Output = $\text{scale} \odot x + \text{bias}$

This gives each channel a learned, input-dependent scaling — the network can dynamically emphasize or suppress features based on the global position.

## KataGo-style global pooling

At blocks 2 and 5, an additional **global pooling branch** injects board-wide context:

1. A parallel 3×3 conv computes `pool_channels` features
2. Three pooling operations: mean, scaled mean, max
3. The pooled features are projected to a per-channel bias added before the second convolution

This lets the network reason about global properties (material balance, king safety, piece counts) while still operating with local 3×3 convolutions.

## Heads

- **Main policy**: `1x1 conv (144 -> 4) -> BN -> ReLU -> flatten -> linear -> ~4206 logits`
- **Terminal value**: `1x1 conv (144 -> 32) -> BN -> ReLU -> global average pool -> MLP -> 3 WDL logits`
- **MLH**: `1x1 conv (144 -> 16) -> BN -> ReLU -> global average pool -> MLP -> 1 scalar`
- **STV**: `1x1 conv (144 -> 32) -> BN -> ReLU -> global average pool -> MLP -> 3 WDL logits`
- **Auxiliary policy**: `1x1 conv (144 -> 2) -> BN -> ReLU -> flatten -> linear -> ~4206 logits`

The two policy heads use the same deterministic move table as the engine. During training, the main policy target comes from the root visit-count distribution, while the auxiliary policy predicts the opponent's reply distribution. The value heads predict both terminal WDL and a short-horizon WDL target; MLH predicts remaining plies to game end.

## Training loss

The trainer uses a weighted multi-head objective:

$$
\mathcal{L} =
1.5 \cdot L_{\text{wdl}} +
1.0 \cdot L_{\text{policy}} +
0.3 \cdot L_{\text{mlh}} +
0.15 \cdot L_{\text{stv}} +
0.15 \cdot L_{\text{aux-policy}}
$$

with optimizer-applied weight decay (`l2_regularization = 3e-5`) outside the loss module. The terminal WDL target also uses light label smoothing (`0.05`).

## Architecture parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `num_residual_blocks` | 8 | Depth of the residual tower |
| `num_filters` | 144 | Width of convolutional layers |
| `se_channels` | 32 | SE bottleneck width |
| `global_pool_channels` | 32 | Global pooling feature width |
| `global_pool_blocks` | (2, 5) | Which blocks get global pooling |
| `policy_channels` | 4 | Conv channels in the main policy head |
| `aux_policy_channels` | 2 | Conv channels in the auxiliary policy head |
| `value_channels` | 32 | Conv channels in the terminal value head |
| `board_channels` | 22 | Input tensor channels |
