The WASM bindings let you run the full hexchess engine in the browser — including AI move search with a neural network model.

## Installation

Install the prebuilt package from npm:

```bash
npm install hexchess-zero
# or
pnpm add hexchess-zero
yarn add hexchess-zero
```

The package ships the compiled WASM binary alongside the JS loader. Import it and initialize before use:

```js
import init, { Game, AiPlayer } from "hexchess-zero";

await init(); // initialize WASM module
```

Works in any modern browser, and in bundlers (Vite, webpack, Next.js) that support WebAssembly imports. For Node.js, use `import init from "hexchess-zero"` with a Node-compatible target.

### Building from source

If you want to hack on the engine, build the WASM package locally from the repo:

```bash
wasm-pack build --target web bindings/wasm
```

This produces a `bindings/wasm/pkg/` directory you can import directly.

## Game class

### Create a new game

```js
const game = new Game();
```

### Get legal moves

```ts
const moves = game.legalMoves();
// [{from_q: -4, from_r: -1, to_q: -4, to_r: 0, promotion: null, notation: "b1-b2"}, ...]
```

Each move is a typed `JsMove` with `from_q`, `from_r`, `to_q`, `to_r`, `promotion`, and a pre-computed Glinski `notation` string.

### Apply a move

```ts
// Preferred: pass a Move from legalMoves(), or Glinski notation directly.
game.apply(moves[0]);
game.applyNotation("f5-f6");
game.applyNotation("f10-f11=Q");  // promotion

// Low-level positional form is still supported:
game.applyMove(0, -4, 0, -3, null);
game.applyMove(0, 4, 0, 5, "queen");
```

These methods throw if the move is illegal.

### Undo and clone

```ts
game.undoMove();            // throws if no moves to undo
const branch = game.clone(); // deep copy for analysis variations
```

### Query game state

```ts
game.sideToMove();  // "white" or "black"
game.status();      // "ongoing", "checkmate_white", etc.
game.isGameOver();  // boolean
game.isInCheck();   // boolean
game.moveCount();   // number of half-moves played
```

### Get board state

```ts
const pieces = game.boardState();
// [{q: -5, r: 0, piece: "rook", color: "white", square: "a1"}, ...]
```

Each `JsPiece` has `q`, `r`, `piece` (king/queen/rook/bishop/knight/pawn), `color` (white/black), and a `square` string in Glinski notation.

### Board encoding

For browser-side ML, you can also pull out the raw tensor:

```ts
import { tensorShape } from "hexchess-zero";

const flat = game.encodeBoard();  // Float32Array, length = 22 * 11 * 11
const [c, h, w] = tensorShape();  // [22, 11, 11]
```

Like the Python bindings, this tensor is encoded in **side-to-move frame** so the network always sees the current player as "my side". The extra metadata planes include repetition, in-check, last-move from/to, and plies-since-pawn-move.

## AiPlayer class

### Heuristic AI (no model needed)

```ts
const ai = new AiPlayer(500); // 500 MCTS simulations per move
const move = ai.bestMove(game);
```

Higher simulation counts produce stronger play but take longer. You can also tune the sim count after construction via `ai.simulations = 200`.

### Full search result

`search()` returns the full MCTS result — best move, scalar value, visit-count policy, and nodes searched:

```ts
const result = ai.search(game, 0.0);
result.best_move;  // JsMove
result.value;      // number in [-1, 1], positive = good for side to move
result.nodes;      // number of nodes searched
result.policy;     // number[] of length numMoveIndices()
```

### Neural network AI

Load an ONNX model file and pass the bytes:

```ts
const response = await fetch("/models/latest.onnx");
const modelBytes = new Uint8Array(await response.arrayBuffer());

const ai = AiPlayer.withModel(500, modelBytes);
const move = ai.bestMove(game);
```

The WASM bindings use [Tract](https://github.com/sonos/tract) for inference — a pure-Rust ML runtime that compiles to WASM. No server-side inference needed. Model loading is synchronous and will block the thread, so for smooth UIs load it inside a Web Worker.

### Temperature + Dirichlet noise

```ts
// temperature=0: always pick the best move (greedy)
// temperature=1: sample proportional to visit counts
const move = ai.bestMoveWithTemperature(game, 0.3);

// Root-move Dirichlet noise (for self-play exploration)
ai.setDirichlet(0.25, 0.3);
ai.setDirichlet(0.0, 0.3);  // disable
```

### Transposition table stats

```ts
const tt = ai.ttStats();
tt.hits; tt.misses; tt.clears; tt.current_size;
```

## Notation helpers

Moves and board cells use `(q, r)` coordinates throughout the API. To display them with human-readable [Glinski notation](/docs/rules#notation) — for move history, UI labels, PGN-style export, etc. — use the built-in helpers:

```ts
import { toNotation, fromNotation, numMoveIndices } from "hexchess-zero";

toNotation(0, 0);     // "f6"  (center)
toNotation(1, -5);    // "g1"  (white king start)

fromNotation("f6");   // { q: 0, r: 0 }
fromNotation("g1");   // { q: 1, r: -5 }
fromNotation("xx");   // throws

numMoveIndices();     // ~4206 — policy vector length
```

## Full example

```ts
import init, { Game, AiPlayer } from "hexchess-zero";

await init();

const game = new Game();
const ai = new AiPlayer(500);

while (!game.isGameOver()) {
  const move = ai.bestMove(game);
  console.log(`${game.sideToMove()} plays ${move.notation}`);
  game.apply(move);
}

console.log(`Game over: ${game.status()}`);
```
