The Python bindings give you full access to the engine, MCTS search, board encoding, and minimax — everything needed for training and analysis.

## Installation

Install the prebuilt package from PyPI:

```bash
pip install hexchess-zero
# or
uv add hexchess-zero
```

Wheels are available for Linux, macOS, and Windows on Python 3.9–3.13. The distribution is named `hexchess-zero` but imports as `hexchess`:

```python
import hexchess
```

### Building from source

If you want to hack on the engine itself, clone the repo and build the bindings locally:

```bash
make setup
# or manually:
uv sync --group dev
uv run maturin develop --release -m bindings/python/Cargo.toml
```

## Game class

### Create and play

```python
import hexchess

game = hexchess.Game()

moves = game.legal_moves()
# [Move(b1-b2), Move(b1-b3), Move(c1-d2), ...]

# Apply a move — accepts a Move from legal_moves() or Glinski notation.
game.apply(moves[0])
game.apply("f5-f6")
game.apply("f10-f11=Q")

# Low-level positional form is still supported:
game.apply_move(0, -4, 0, -3)

game.side_to_move()   # "white" or "black"
game.status()         # "ongoing", "checkmate_white", etc.
game.is_game_over()   # bool
game.is_in_check()    # bool
game.move_count()     # number of half-moves played
```

### Move objects

Each entry in `legal_moves()` is a `hexchess.Move` with typed attributes:

```python
mv = game.legal_moves()[0]

mv.from_q, mv.from_r   # axial coordinates of the from-square
mv.to_q, mv.to_r       # axial coordinates of the to-square
mv.from_, mv.to        # (q, r) tuples
mv.promotion           # "queen" / "rook" / "bishop" / "knight" / None
mv.notation            # "b1-b2"  (Glinski notation)

# Moves are hashable and comparable
{mv, mv} == {mv}       # True

# Construct from notation
mv = hexchess.Move.from_notation("f10-f11=Q")
```

### Board state and undo

```python
pieces = game.board_state()
# [Piece(q=-5, r=0, piece="rook", color="white"), ...]
for p in pieces:
    print(p.square, p.piece, p.color)  # "a1 rook white"

game.undo_move()  # undo the last move

# Deep copy for branching analysis
import copy
game2 = copy.deepcopy(game)

# Print an ASCII rendering of the board — useful in notebooks.
print(game)
```

### Pawn promotion

```python
game.apply("f10-f11=Q")                # via notation
game.apply_move(0, 4, 0, 5, "queen")   # via positional API
# promotion piece: "queen", "rook", "bishop", "knight"
```

## MCTS search

### Heuristic search (no model)

```python
mcts = hexchess.MctsSearch(simulations=500)

result = mcts.run(game)
result.best_move       # hexchess.Move
result.value           # float in [-1, 1], positive = good for side to move
result.policy          # numpy array of shape (num_move_indices,)
result.nodes           # number of nodes searched
```

### Neural network search

```python
mcts = hexchess.MctsSearch(
    simulations=800,
    model_path="path/to/model.onnx",
    c_puct=1.5,
    batch_size=32,
    tt_capacity=500_000,
)

result = mcts.run(game)
game.apply(result.best_move)
```

### Temperature and Dirichlet noise

```python
# Temperature controls move sampling: 0 = greedy, 1 = proportional to visits.
result = mcts.run(game, temperature=1.0)

# Dirichlet noise is configured at construction time (used for self-play).
explorer = hexchess.MctsSearch(
    simulations=800,
    dirichlet_epsilon=0.25,
    dirichlet_alpha=0.3,
)

# `simulations` is a read/write property.
explorer.simulations = 200
```

### Transposition table stats

```python
stats = mcts.tt_stats()
stats.hits, stats.misses, stats.clears, stats.current_size
```

## Minimax search

For classical alpha-beta search (used in imitation bootstrap and analysis):

```python
# Basic minimax
result = hexchess.minimax_search(game, depth=5)
result.best_move       # hexchess.Move
result.score           # centipawn score (positive = good for side to move)
result.nodes           # nodes searched

# Search with scores for ALL legal moves
result = hexchess.minimax_search_all(game, depth=5)
for ranked in result.moves:
    print(ranked.move.notation, ranked.score)

# Two-phase search: deep best-move + shallow policy for all moves
result = hexchess.minimax_search_with_policy(game, depth=5)
result.best_move
result.best_score
result.moves           # list[RankedMove]
```

### Custom evaluation weights

```python
weights = hexchess.EvalWeights(
    material=1,
    mobility=0,
    pawn_advance=1,
    center_control=4,
    king_safety=0,
    bishop_color_bonus=30,
    pawn_connected=7,
    pawn_isolated=-10,
    passed_pawn=3,
    king_tropism=2,
)
result = hexchess.minimax_search(game, depth=5, weights=weights)

# Or material-only evaluation:
weights = hexchess.EvalWeights.material_only()
```

## Board encoding (for training)

Convert game states to tensors for neural network training:

```python
# Single position → numpy array of shape (22, 11, 11)
tensor = hexchess.encode_board(game)

# Batch encoding → numpy array of shape (N, 22, 11, 11)
tensors = hexchess.encode_batch([game1, game2, game3])

# Shape + policy-size constants
hexchess.TENSOR_SHAPE       # (22, 11, 11)
hexchess.NUM_MOVE_INDICES   # ~4206
```

The encoder is **side-to-move framed**: when Black is to move, the board is mirrored and piece colors are swapped so the network always sees "my" pieces and "my" pawn direction.

The 22 channels encode:
- Channels 0-5: side-to-move piece planes
- Channels 6-11: opponent piece planes
- Channel 12: absolute side to move
- Channel 13: fullmove number
- Channel 14: halfmove clock
- Channel 15: en passant
- Channel 16: repetition count
- Channel 17: validity mask (which cells are valid hex cells)
- Channel 18: in-check flag
- Channel 19: last-move from square
- Channel 20: last-move to square
- Channel 21: plies since pawn move

## Move indexing

Convert between moves and policy vector indices:

```python
# Absolute-frame move → index
idx = hexchess.move_to_index(0, -4, 0, -3)

# Preferred for training targets paired with encode_board():
mv = game.legal_moves()[0]
stm_idx = game.policy_index(
    mv.from_q, mv.from_r,
    mv.to_q, mv.to_r,
    mv.promotion,
)

# Index → move
move = hexchess.index_to_move(idx)  # hexchess.Move(b1-b2)

# Total policy vector size
hexchess.num_move_indices()  # ~4206
```

`move_to_index()` is an absolute-frame mapping. `Game.policy_index()` applies the Black-to-move mirror remap so the target matches the STM-framed tensor emitted by `encode_board()`.

The move index table is deterministic — any change would break compatibility with existing training data and models.

## Notation helpers

Moves and board cells use `(q, r)` coordinates throughout the API. To display them with human-readable [Glinski notation](/docs/rules#notation) — for move history, UI labels, PGN-style export, etc. — use the built-in helpers:

```python
hexchess.to_notation(0, 0)      # "f6"  (center)
hexchess.to_notation(1, -5)     # "g1"  (white king start)

hexchess.from_notation("f6")    # (0, 0)
hexchess.from_notation("g1")    # (1, -5)
```

`from_notation` raises `ValueError` on malformed or off-board input.
