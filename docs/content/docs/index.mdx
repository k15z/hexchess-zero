Hexchess Zero is a hexagonal chess engine implementing the [Glinski variant](https://en.wikipedia.org/wiki/Hexagonal_chess#Gli%C5%84ski's_hexagonal_chess) — chess played on a 91-cell hexagonal board. The core engine is written in Rust and provides:

- **Complete Glinski rules** — move generation, check/checkmate/stalemate detection, draw conditions (threefold repetition, 50-move rule, insufficient material)
- **AlphaZero-style MCTS** — Monte Carlo Tree Search with neural network evaluation for strong play
- **TypeScript / WASM bindings** — play in the browser with a bundled AI opponent
- **Python bindings** — full access to the engine, MCTS, board encoding, and minimax search for training and research

## Architecture

The project is a monorepo with the Rust engine as the single source of truth for game rules:

```
engine/          Rust engine — board, move generation, MCTS, inference
bindings/wasm/   WASM bindings for browser play (tract inference)
bindings/python/ PyO3 bindings for training (ONNX Runtime inference)
training/        AlphaZero distributed training pipeline
docs/            Documentation site with interactive playground
```

The engine compiles to three targets:

| Target | Inference backend | Use case |
|--------|------------------|----------|
| Native (Rust) | ONNX Runtime | Testing, benchmarks |
| WASM | Tract | Browser play |
| Python (PyO3) | ONNX Runtime | Training pipeline |

## Install

Prebuilt packages are published under the name **`hexchess-zero`**:

```bash
pip install hexchess-zero       # Python — imports as `hexchess`
npm install hexchess-zero       # TypeScript / WASM
```

Python wheels cover Linux, macOS, and Windows on Python 3.9–3.13. The npm package bundles the WASM binary and JS loader, so no separate build step is needed. See [Python](/docs/usage/python) or [TypeScript](/docs/usage/typescript) for API usage.

## Developing locally

```bash
# Run engine tests
make test

# Documentation site with interactive playground
make docs-dev

# Build Python bindings from source (for training / engine work)
make setup
```

See the [Usage](/docs/usage) section for TypeScript and Python API guides, or the [Research](/docs/research) section for details on the training pipeline and MCTS implementation.
